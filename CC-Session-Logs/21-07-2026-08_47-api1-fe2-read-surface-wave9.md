## Key Learnings
- **A sampled compile-time contract guard is FALSE SECURITY** (the session's biggest lesson). FE-2's first `decimal-contract.ts` asserted only 8 representative governed decimals were `string`; the adversarial finder proved (in a scratch git worktree, mutating the generated types) that changing `VarRowOut.sigma` to `number` compiled clean — ~35 displayed decimals were unguarded, reopening the exact FL-1 "governed number wrong in the UI" class. Fix: an EXHAUSTIVE guard (`OnlyCountsAreNumbers` over all 14 `*RowOut`: the ONLY fields ever typed `number` may be a curated `CountKey` union of 17 integer counts). Rule: a contract guard must be exhaustive over the generated schema, or explicitly re-verified complete each slice.
- **The stage10 filename zero-pad hazard (CC-2-recorded):** a two-digit `stage10` suite sorts lexically BEFORE `stage2`/`stage4`, so in a single-invocation shared-DB local PG battery it would seed extra runs before earlier stages assert their exact count pins. Solution: name the suite `test_demo_stage9z_api1_reads*` so it collates LAST (after `stage9_cc2`). Empirically confirmed with `LC_ALL=C sort`.
- **Demo stage-10 prereq subtleties (surfaced only via live PG validation):** (a) active-risk needs the covariance run over EXACTLY `{FX_USD, FX_EUR}` — a bare "latest covariance" picked a currency-less later-stage (RS-1 multifamily) matrix the binder refuses; (b) the proxy factor-exposure family admits CURRENCY factors ONLY, so it must run over the campaign's DEMO-GLOBAL book (its sole proxied private PE-HARBOR-IV→FX_USD is currency), never a RATES-carrying RS-1-sleeve exposure run; (c) resolve all existing evidence-run ids BEFORE any run write so run 5's own proxy run doesn't shadow the allocation run resolved for runs 2/3.
- **Drift-check determinism must be verified across CI vs local Python** — the correctness finder rebuilt a fresh Python-3.12 venv (CI uses 3.12, local was 3.13) with pinned pydantic==2.13.4/fastapi==0.137.1 + `sort_keys=True` dump + `npm ci` → byte-identical `openapi.json`, so the drift-check won't false-positive on its own PR.
- **The audit writer's `_iso` treats naive datetimes as UTC; a reader must mirror that** — FastAPI parses an offset-less `since`/`until` query param to a naive `datetime`, and `astimezone()` on a naive dt assumes LOCAL time → an off-UTC-server compliance-window bug.
- **openapi-typescript maps `{type:string}`→`string` and `anyOf[number,string]`→`number|string`** — so governed response decimals (declared `str` in every `*RowOut`) generate to `string` automatically; only request `*In` `Decimal` fields (Pydantic default `anyOf`) carry the union. The precondition that could have sunk FE-2 was pre-empted before implementation.
- **Route ordering is load-bearing:** `GET /{family}/latest` must be declared BEFORE `GET /{family}/{result_id}` (else the UUID path-param shadows `latest`). Verified for every new latest/by-id pair.
- **Per-family `/latest` required-key shapes differ:** benchmark-relative keys on `portfolio_id` alone; desmoothed-returns needs `portfolio_id`+`instrument_id` (a test-writing gotcha).

## Solutions & Fixes
- **Shared read helper (`calc/reads.py`):** `list_governed_results(session, model, *, acting_tenant, filters, run_type, as_of, order_by)` joins `CalculationRun`, filters tenant + COMPLETED, str-coerces entity filters when not None, `system_from <= as_of`, orders `system_from DESC, run_id DESC, grain ASC`; `order_by` generalized to accept a single column OR a tuple (covariance canonical pair order). `latest_run_rows(rows)` keeps only the newest run's rows. Factors CC-2's pacing latest-resolver.
- **var/es-backtest share `var_backtest_result`** — the entity reads pass `run_type=RUN_TYPE_VAR_BACKTEST`/`RUN_TYPE_ES_BACKTEST` to disambiguate.
- **Audit `_iso` naive-UTC fix (`audit/queries.py`):** mirror the FROZEN writer's guard — `if dt.tzinfo is None: dt = dt.replace(tzinfo=UTC)` before `astimezone(UTC).isoformat()`. Regression-tested via a unit assertion `_iso(naive) == _iso(UTC)`.
- **Covariance factor-set-matched resolver (`stage10_api1.py` `_resolve_covariance_run_over`):** unions `factor_id_1` + `factor_id_2` per COMPLETED covariance run and returns the run whose set == `{fx_usd, fx_eur}`.
- **DEMO-GLOBAL exposure-run scoping (`_resolve_exposure_run_for_portfolio`):** joins `ExposureAggregate.portfolio_id == demo_global_id` so the proxy run is over the currency-only campaign book.
- **The FL-1 kill (`types.ts`):** `FamilyColumn<F> = { key: keyof FamilyRowOut[F] & string; label: string }` + `FAMILY_ROW_COLUMNS: { [F in Family]: FamilyColumn<F>[] }` — a bogus column key fails `tsc` TS2322 (proven, reverted).
- **Exhaustive decimal guard (`decimal-contract.ts`):** `type NumberKeys<T> = { [K in keyof T]-?: number extends NonNullable<T[K]> ? K : never }[keyof T]`; `OnlyCountsAreNumbers<T> = Exclude<NumberKeys<T>, CountKey> extends never ? true : false`; `AssertTrue<T extends true>` over all 14 `*RowOut`. Proven to bite (`TS2344: Type 'false' does not satisfy the constraint 'true'`).
- **CI drift-check job** ("API type drift"): dedicated `setup-python` + `setup-node` job (Frontend job is Node-only) → `python scripts/dump_openapi.py` → `npm run -w apps/frontend gen:types` → `git diff --exit-code apps/frontend/openapi.json apps/frontend/src/api/generated`. Mirrored locally by `make gen-api-check`.
- **Generated dir ignored from lint/format** (`.prettierignore` + eslint `ignores` add `src/api/generated`), still typechecked by tsc (`.d.ts` under `include:["src"]`, `skipLibCheck:true`).
- **`scripts/dump_openapi.py`:** offline `app.openapi()` → `apps/frontend/openapi.json` with `json.dumps(..., sort_keys=True, indent=2)+"\n"` for deterministic diffs.
