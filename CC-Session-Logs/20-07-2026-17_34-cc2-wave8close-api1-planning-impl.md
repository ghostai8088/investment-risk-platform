## Key Learnings
- **A compounding kernel is a new hazard class:** pacing NAV compounds geometrically and can overflow the Decimal context; a post-create magnitude gate is not enough on its own — the KERNEL needs its own runaway ceiling. The bounded-stat governed numbers never needed this.
- **The magnitude envelope MUST sit strictly BELOW the result column capacity:** `PreciseDecimal(28,6)` holds < 1e22, so `1E21` (the var/es-backtest `_MAX_RESULT_ABS` precedent) — a value in [1e22,1e26) passes a looser gate then raises a **PG numeric overflow that is INVISIBLE at the SQLite test tier** (AD-011 engine split). A PG-tier FAILED-path test is the catch.
- **The pre-ratification verifier pass keeps earning its keep:** it caught CC-2's period-geometry HIGH pre-implementation, and for API-1 it REFUTED the flagship VaR read's read-only feasibility (exposure/factor-exposure runs are portfolio-SUBTREE-scoped; neither `calculation_run` nor `dataset_snapshot` records the run's ROOT portfolio) — reshaping the scope before it baked in.

- **Route ordering:** `GET /{family}/latest` must be declared BEFORE `GET /{family}/{result_id}` (else "latest" fails uuid parsing on the by-id route → 422, never reaching /latest).
