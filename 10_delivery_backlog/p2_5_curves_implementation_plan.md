> **What P2-5 is — and is NOT.** P2-5 **captures** vendor curves the way P2-2/P2-4 captured FX rates / prices: an FR bitemporal `curve` header (create / effective-dated supersede / as-known correction / `reconstruct_curve_as_of`) owning an immutable, version-pinned set of `curve_point` nodes `(tenor, value_type, value)`, governed by `MARKET.CURVE_*` audit + VENDOR `data_source` lineage + a fail-closed DQ gate, under symmetric tenant RLS. It computes **nothing**: **NO curve-construction model, NO interpolation, NO bootstrapping, NO discounting engine, NO duration / key-rate / spread-duration, NO pricing/valuation model, NO return/factor/covariance/VaR/ES, NO exposure recalculation, NO `calculation_run` change, NO `dataset_snapshot` schema change.** It is the captured **curve input** future calculations will consume; the snapshot/run binding is **readiness-only** (the P2-3 `COMPONENT_KIND_FX` precedent — pin when a calc needs it).

## 1. Requirements included
P2-5 touches **TWO** backbone requirements (review fold — product/qa #1; the curve domain is split across them):
- **REQ-PUB-002** — "Curves & volatility surfaces" (CAP-3.2/3.3; realizing entities `yield_curve`/`volatility_surface` FR). P2-5 realizes the **captured yield curve** (`curve` + `curve_point`) + the **"curve values reproduce"** acceptance leg (capture/supersede/correct/`reconstruct_curve_as_of` on both axes). **Advances to In-Progress (partial)** — does **NOT close**: **`volatility_surface`** (ENT-022, P3+ options risk) and the **"interpolation method declared / QS-13 interpolation-test"** leg are **DEFERRED** (the partial advance rests **solely** on the curve-values-reproduce leg; capturing an inert `interpolation_method` label per OQ-P2-5-9 does **NOT** satisfy the "method declared" leg — that needs the QS-13 interpolation test P2-5 does not build).
- **REQ-PUB-003** — "Credit spreads, ratings, benchmarks" (CAP-3.4/3.5; realizing entities `credit_spread`/`rating`/`benchmark`). P2-5 realizes **`credit_spread` (ENT-023) capture** (`curve_type=CREDIT_SPREAD` over the SAME `curve`/`curve_point` tables) + the **"inputs present & as-of"** spread-coverage leg. **Advances to In-Progress (partial)** — does **NOT close**: `rating` + `benchmark` (P2-6+) + the full Coverage test are **DEFERRED**. *(`credit_spread` is REQ-PUB-003's entity, NOT REQ-PUB-002's — the canonical backbone split.)*

## 2. Requirements excluded (hard)
**NO curve-construction model**; **NO interpolation / bootstrapping / discounting engine**; **NO duration / key-rate duration / spread-duration**; **NO pricing / valuation model**; **NO return / performance analytics**; **NO factor / covariance model**; **NO VaR / Expected Shortfall / risk / sensitivities / stress / scenario**; **NO exposure recalculation / `exposure_aggregate` change**; **NO `calculation_run` wiring change**; **NO `dataset_snapshot` schema redesign**; **NO `fx_rate` / `price_point` schema redesign**; **NO `volatility_surface` (ENT-022)**; **NO `rating` / `benchmark` (P2-6+)**; **NO market-data feed-ingestion pipeline**; **NO corporate-action adjustment engine**; **NO reporting/dashboard / frontend**; **NO P3+**; **NO broad DQ/lineage rewrite**; **NO `audit/service.py` change**; **NO BYPASSRLS**; **NO hybrid/SYSTEM_TENANT** (curve data is per-tenant vendor-licensed). **REQ-PUB-002 AND REQ-PUB-003 advance to In-Progress (partial)** (§1 — the curve-values-reproduce + spread-coverage legs met; vol-surface, interpolation-test, rating/benchmark, and the full Coverage test deferred) — neither closes; **no other REQ status changes.**

## 3. Proposed entities / modules
- **`curve`** (ENT-021 header, **FR**) + **`curve_point`** (ENT-021 "Curve nodes", **IA append-only version-pinned children**) — the two new domain tables; **a new migration `0020_curve`** (down_revision `0019_price_point`): `curve` FR + symmetric RLS, NOT append-only (mirrors `0017_fx_rate`/`0019_price_point`); `curve_point` symmetric RLS + **in `APPEND_ONLY_TABLES`** (the `irp_prevent_mutation` P0001 trigger + ORM guard — the `dataset_snapshot_component`/`transaction` precedent). All promoted key columns DB-level **NOT NULL**. The `curve` header also carries a **nullable inert `interpolation_method` metadata String** (OQ-P2-5-9, APPROVED) — captured-but-unused (NO interpolation engine, NO DQ rule on it, NOT in the unique key; the scope-fence asserts no interpolation symbol consumes it), plus `point_count` (the DC-2 audit metadata).
- **`irp_shared/marketdata` extension (additive):** `Curve` + `CurvePoint` models (in `marketdata/models.py` or a new `marketdata/curve.py`); the curve binder (`capture_curve` / `supersede_curve` / `correct_curve` / `reconstruct_curve_as_of` / `resolve_curve` + the points written-with-the-header) + the DQ gate; `MARKET.CURVE_*` constants + `CurveActor` + `ensure_vendor_source(VENDOR_CURVE)` + the ORIGIN edge (in `marketdata/curve.py`/`events.py`). The package's leaf one-way imports `{reference, lineage, dq, audit, db}` are unchanged; imports NO `calc`/`exposure`/`snapshot` symbol.

- **New backend `api/marketdata.py` endpoints** (extend the existing file): the curve capture/supersede/correct/read endpoints (§15).
- **`entitlement/bootstrap.py`:** **UNCHANGED** — `marketdata.view`/`.ingest` already exist; only the parity test extends.

## 4. Temporal classification
**`curve` = FR / fully-reproducible / bitemporal** (`FullReproducibleMixin`; `__temporal_class__ = FULL_REPRODUCIBLE`; **NOT** in `APPEND_ONLY_TABLES`; close-out UPDATEs; content-immutability service-enforced + tested) — the `fx_rate`/`price_point` protocol. **`curve_point` = IA append-only** (`ImmutableAppendOnlyMixin`; in `APPEND_ONLY_TABLES` → the `irp_prevent_mutation` P0001 trigger + ORM guard) — immutable nodes pinned to one `curve` physical version (`curve_id`); a re-version creates a NEW header + a fresh point set, never an edit. The **curve-as-a-whole is FR-reconstructable** (`reconstruct_curve_as_of` returns the header version, whose `curve_id` reads the pinned points). *(OQ-P2-5-1: points-independently-FR alternative, rejected for v1.)*

## 5. Curve grain — **recommended: `(curve_type, currency_code, reference_key, curve_date, curve_source)`**
**Recommendation + rationale.** The v1 current-head partial-unique is **`(tenant_id, curve_type, currency_code, reference_key, curve_date, curve_source) WHERE valid_to IS NULL AND system_to IS NULL`** — exactly one open curve per that 6-tuple; the FR axes version the curve for the fixed key. `curve_source` is in the key (multi-vendor coexistence, the `price_source` precedent). `reference_key` is in the key with sentinel `"NONE"` for rate curves (OD-P2-5-K), so credit spreads and rate curves share one uniform NOT-NULL key. Each header row is independently traceable to one VENDOR `data_source` ORIGIN edge; its `curve_id` owns the node set.

## 6. Curve point grain
**`(curve_id, value_type, tenor_days)`** UNIQUE within a curve version — one value per (curve version × value_type × normalized tenor). Columns: `curve_id` (NOT-NULL FK → `curve.id`), `tenor_label` (String), `tenor_days` (Integer, the key, `>0`), `value_type` (String, OD-P2-5-H), `point_value` (`Numeric(20,12)`), `tenant_id` (RLS, carried from the header). Plain UNIQUE (immutable children, not a current-head partial-unique). A curve version may carry several `value_type`s and many tenors.

## 7. Curve date / valid-time convention
`curve_date` is a **separate immutable logical-key `Date`** (the business date the curve is FOR), carried forward verbatim, **never mutated**, DISTINCT from the FR `valid_from` axis. `valid_from`/`valid_to` (valid-time) + `system_from`/`system_to` (knowledge-time) version the *curve* for a fixed `curve_date`. `reconstruct_curve_as_of(valid_at, known_at)` reads the curve header true-at `valid_at` as-known-at `known_at` for a logical key, then its pinned points (the `reconstruct_*_as_of` shape).

## 8. Curve type / curve family convention
`curve_type` controlled-vocab plain String (no enum/CHECK; app-side allow-list — the `price_type` shape). v1 **`{TREASURY, GOVT, SWAP, OIS, CREDIT_SPREAD}`**: TREASURY/GOVT/SWAP/OIS = rate/yield families; CREDIT_SPREAD = the spread family (with `reference_key` carrying the issuer/rating/sector dimension). An out-of-vocab `curve_type` → `CurveValueError` (422). `MUNI`/`BASIS`/`INFLATION`/`volatility_surface` reserved, not minted.

## 9. Tenor convention
A canonical **`tenor_label`** String (`{N}{D|W|M|Y}`, e.g. `"1M"`/`"2Y"`/`"30Y"`) + a normalized **`tenor_days`** Integer (the key, strictly positive). Key on `tenor_days` (the normalized form) so `"12M"`/`"1Y"` cannot double-insert. v1 validation: `tenor_days > 0` (a value check + the DQ tenor-validity rule); `tenor_label` matches the `{N}{D|W|M|Y}` pattern (else `CurveValueError` 422). The label→days mapping is a **deterministic captured normalization** (D=1/W=7/M=30/Y=365, recorded), **not** a day-count or interpolation engine. **NO interpolation between tenors.**

## 10. Value convention — yield rate / spread / units / scale
`point_value` = a **canonical DECIMAL fraction** (NOT percent, NOT bps): 4.25% → `0.042500000000`; 15bp spread → `0.001500000000`; a discount factor → `0.973…`. `value_type` controlled-vocab **`{ZERO_RATE, PAR_RATE, DISCOUNT_FACTOR, SPREAD}`** (what `point_value` means). Scale **`Numeric(20,12)`** (rate precision; the `fx_rate`-family high-precision scale, NOT money scale 6). **Sign per `value_type`:** DISCOUNT_FACTOR strictly-positive; ZERO_RATE/PAR_RATE/SPREAD may be **negative** (negative rates/spreads are real). **No interpolation/bootstrapping/discounting math** — `point_value` is captured verbatim. (OQ-P2-5-2 scale; OQ-P2-5-4 bounds.)

## 11. Currency convention
Captured `currency_code` (ISO String(3)), validated via the reused hybrid-aware `resolve_currency` (own OR SYSTEM), **in the header key**. The curve is stored in its native currency; **NO conversion** in P2-5. No `convert` import in the curve binder.

## 12. Source / provenance model
`curve_source` a controlled-vocab String label (e.g. `BLOOMBERG`/`REFINITIV`/`MARKIT`) — a **key** component — plus a **VENDOR `data_source` ORIGIN lineage edge** per new physical curve version via a new `ensure_vendor_source(VENDOR_CURVE)` (the `VENDOR_FX`/`VENDOR_PRICE` sibling; REUSES `record_lineage` UNCHANGED), targeting the `curve` header row. **No market-data feed-ingestion pipeline / connector / scheduler** — captures arrive through the governed binder.

## 13. Relationship to `dataset_snapshot`
**READINESS ONLY** — P2-5 mints **NO `COMPONENT_KIND_CURVE`** (it stays RESERVED in `SNAPSHOT_COMPONENT_KINDS`) and changes **NO `build_snapshot`/`_reresolve_content`**. A curve component is pinned **only** when a future calculation consumes a curve (the P2-3 `COMPONENT_KIND_FX` precedent). **NO `dataset_snapshot` schema redesign** (the tables are untouched).

## 14. Relationship to `calculation_run`, if any — **expected: none in P2-5**
**NONE.** Captured curve data wires no `calculation_run`; `calc`/`exposure`/`snapshot` are **untouched**; `exposure_aggregate` is **unchanged**. Binding a curve into a run (discounting, key-rate risk) is a future *calculation* slice (P3+), explicitly out of P2-5.

## 20. Data-quality behavior
Two layers, both fail-closed:
- **Binder-side value checks (`CurveValueError` → 422, before any write; NOT the DQ Protocol):** out-of-vocab `curve_type`/`value_type`; out-of-pattern `tenor_label`; and the **`curve_type` ↔ `reference_key` invariant** (review fold — data-arch #2): `curve_type` in `{TREASURY,GOVT,SWAP,OIS}` ⇒ `reference_key == "NONE"`; `curve_type == CREDIT_SPREAD` ⇒ `reference_key != "NONE"` and non-empty (keeps the 6-tuple current-head key unambiguous — the sentinel is a checked invariant, not a convention).
- **Governed DQ gate (co-transactional; `DATA.VALIDATE`; CTRL-032 rollback):** **required-field NOT_NULL** (header: `curve_type`/`currency_code`/`reference_key`/`curve_date`/`curve_source`; each node: `tenor_days`/`value_type`/`point_value`) + **tenor validity** (`tenor_days > 0`, a `RANGE` `{min:0, min_inclusive:false}`) + the **value-type-conditional `RANGE`** on `point_value`. **Mechanism (review fold — data-arch/qa/architect/lineage-dq):** the shipped `dq/rules.py:evaluate_range` is **single-column** with one fixed `min`/`max` and cannot branch on a sibling `value_type`, so the conditional gate is realized **binder-side as TWO `RANGE` registrations**, each run via `run_quality_check` over a **`value_type`-filtered sub-dataset** built in `capture_curve` (the `price.py` `dataset=[{…}]` pattern): (1) **`curve.discount_factor_positive`** `{column:"point_value", min:0, min_inclusive:false}` over the `DISCOUNT_FACTOR` rows; (2) **`curve.rate_spread_sanity`** `{column:"point_value", min:-1, max:1}` over the `ZERO_RATE`/`PAR_RATE`/`SPREAD` rows (negatives allowed; OQ-P2-5-4 band). The `(params, dataset)` **Protocol is UNTOUCHED** (no new evaluator — two ordinary single-column RANGE rules + partitioned datasets; the shipped evaluators behavior-unchanged). **Completeness-by-required-tenor-set + staleness gates DEFERRED** (OQ-P2-5-8).

## 21. Tests
SQLite logic + PG FORCE-RLS + endpoint + parity (the `fx_rate`/`price_point` test shape):
- **FR protocol (positive):** capture (header + nodes) → `reconstruct_curve_as_of` (both axes: a valid-time supersede + an as-known correction reconstruct the right header + its pinned nodes); content-immutability-on-correction (prior header's content columns never mutated — only `valid_to`/`system_to`; the prior version's nodes untouched); the 6-part current-head partial-unique (one open curve per key; many `curve_date`s/sources coexist); multi-vendor coexistence (two `curve_source`s = two open heads).
- **Header/points binding:** a supersede/correction creates a NEW header version + a FRESH node set pinned to the new `curve_id`; the prior version's nodes remain readable + immutable; `curve_point` is append-only (the P0001 trigger + ORM guard reject an UPDATE/DELETE).
- **Tenor + value:** `tenor_days` normalization (`"12M"` and `"1Y"` collide on `tenor_days=365` → uniqueness rejects the dup); out-of-pattern `tenor_label` → `CurveValueError`; out-of-vocab `curve_type`/`value_type` → `CurveValueError`.
- **DQ fail-closed (value-type-conditional — review fold):** a DISCOUNT_FACTOR ≤ 0 → `DataQualityError` → CTRL-032 rollback (no header, no nodes, no ORIGIN edge, no `MARKET.CURVE_CREATE`); a `tenor_days ≤ 0` → fail closed; a missing required field → fail closed; a **negative `ZERO_RATE` / `PAR_RATE` / `SPREAD` is ACCEPTED** (all three negative-allowed value_types, negatives allowed — the value-type-conditional gate); an out-of-band rate/spread (|value|>1) → fail closed. **Cross-partition assertion:** a `DISCOUNT_FACTOR` row is measured ONLY against `curve.discount_factor_positive` (not the rate band) and a rate/spread row ONLY against `curve.rate_spread_sanity` (not DF-positive) — so the two-rule `value_type`-partitioning (§20) is positively tested, not assumed (e.g. a `DISCOUNT_FACTOR = 1.5` passes — DF has no upper bound in v1 — while a `ZERO_RATE = 1.5` is rejected by the sanity band).
- **Curve_type ↔ reference_key invariant (binder-side `CurveValueError` 422 — review fold):** a rate `curve_type` (TREASURY/GOVT/SWAP/OIS) with a non-`"NONE"` `reference_key` → 422 (no write); a `CREDIT_SPREAD` with `reference_key == "NONE"` (or empty) → 422; the happy paths (rate curve + `"NONE"`; `CREDIT_SPREAD` + an opaque label) succeed.
- **Audit:** `MARKET.CURVE_CREATE` (1, header-grained) on capture; `MARKET.CURVE_UPDATE`+`CURVE_CREATE` (2) on supersede; `MARKET.CURVE_UPDATE`+`CURVE_CORRECTION` (2) on correct; `verify_chain` ok (PG); **NO event on read**; **one event per curve regardless of node count** (capture a 1-node curve and a 30-node curve — each emits exactly ONE `MARKET.CURVE_CREATE`; the event count is independent of `point_count` — the load-bearing "points fold into the header write" invariant); `audit/service.py` untouched.
- **Lineage:** one VENDOR (`VENDOR_CURVE`) ORIGIN edge per physical curve version (header-targeted, independent of node count); the close-out roots none; CTRL-032 rollback (monkeypatch the emitter → zero rows).
- **Currency / reference_key:** unknown currency → `CurrencyNotVisible`; `reference_key` is a plain String (a scope-fence test asserts no FK/resolver on it in v1); sentinel `"NONE"` for rate curves vs an opaque label for CREDIT_SPREAD.
- **Entitlement:** deny-by-default 403 (no DB side-effect on denial); the parity test (curve reuses `marketdata.*`); 404/409/422 mapping.
- **PG (`irp_app`):** tenant isolation + no-context → 0 rows on BOTH `curve` + `curve_point`; forged-tenant `WITH CHECK` 42501 on BOTH; the **symmetric-policy + FORCE-RLS shape asserted for BOTH tables** (`qual == with_check`, no `SYSTEM_TENANT` literal, `relforcerowsecurity` true — enumerated per table so the child is not an unpoliced leak); **closed-hybrid-set-unchanged**; `curve_point` append-only P0001 trigger proven (granted UPDATE/DELETE → P0001 raise, not a 42501 privilege denial — the `dataset_snapshot_component` precedent); `curve` FR (no trigger).
