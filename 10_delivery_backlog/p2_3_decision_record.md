## Part 2 — Decision detail

### OD-P2-3-A — `exposure_aggregate` temporal class → **IA, TRUE append-only**
`exposure_aggregate` is **Immutable Append-Only and genuinely immutable** — in `APPEND_ONLY_TABLES`, protected by the `irp_prevent_mutation` **P0001 trigger** + the ORM `before_update`/`before_delete` guard (the `transaction`/`dataset_snapshot` exemplar). A governed derived number is a **fact of a run** — created once, never edited; a re-run produces a **new** run + new rows. (FR rejected — an exposure is never effective-dated/superseded.) **Contrast:** `calculation_run` (ENT-026) is IA-classed but its `status` mutates in place (`update_run_status`), so it is **NOT** in `APPEND_ONLY_TABLES`; the run is the status projection, the exposure rows are the immutable result.

### OD-P2-3-B — exposure grain (v1) → **`(portfolio_id, instrument_id, base_currency)`**
**Decision.** v1 exposure = the **per-holding market-value exposure in base currency**: one `exposure_aggregate` row per `(portfolio_id, instrument_id)` of the bound snapshot, `exposure_amount = signed_quantity × mark_value × fx(mark_currency→base)`, `exposure_type = MARKET_VALUE`.
**Rationale (minimal AND auditable).** This is the **most auditable** grain: each row maps **1:1** to exactly one snapshot POSITION + one VALUATION + the FX component(s) used, so lineage, the DQ gate, and the deterministic recompute are crisp per row. It is still minimal — **no** cross-portfolio aggregation, **no** subtree rollup, **no** netting. A portfolio/node TOTAL is a trivial **deterministic Σ over rows** (a coarser grain is purely additive later) — **deferred (OQ-P2-3-1)** rather than guessed. ("`exposure_aggregate`" = the run-bound **result set**; the per-holding rows are its atoms.) **NOT netting, NOT a risk number.**

### OD-P2-3-C — base currency
**Decision.** The base/reporting currency is a **`calculation_run` parameter** (recorded on the run + every `exposure_aggregate` row's `base_currency`). Default resolution: the bound **portfolio's `base_currency_code`** (an existing EV field) if set, else **`USD`** (the P2-2 `DEFAULT_BASE`). **No new per-tenant/portfolio base-config store** is built (deferred — OD-P2-2-E carried forward). **Rationale.** Reuses existing data; keeps P2-3 to the compute, not a config-management feature; the base is explicit + reproducible (recorded on the run).

### OD-P2-3-D — FX conversion (reproducible, fail-closed) — *single path: pure captured-content read* *(review fold — qa #10)*
**Decision (ONE reproducibility path).** The exposure binder converts each captured `mark_value` (in `mark_currency`) to `base_currency` by reading **ONLY the snapshot's pinned `COMPONENT_KIND_FX` captured content** (the immutable serialized leg rates + base/quote direction + `rate_date`) and applying the P2-2 conversion **arithmetic** (direct → reciprocal → triangulation-through-the-configured-base) **as a PURE function over those captured values**. The compute path **does NOT call `convert(...)`/`reconstruct_fx_rate_as_of(...)` against live FX at run time** — reproducibility comes from the **immutability of the snapshot's captured FX content**, not from an as-of re-read (a re-run over the SAME snapshot is identical even after a later vendor correction; TR-09). The FX is selected/pinned at **snapshot-build** time (inside the *snapshot* package, where `reconstruct_fx_rate_as_of` legitimately picks each leg's as-of version); the *exposure* package's compute is import-fenced from any live/un-pinned FX read. **Exact-date `rate_date == date(as_of_valid_at)`** (v1); a missing leg at build → **`FxRateNotFound`** → the FX-completeness gate **fails closed** (no snapshot ⇒ no run result). **No stale-rate fallback, no silent 1.0, no interpolation.** Because a triangulated path multiplies **two** pinned legs (each possibly a reciprocal), the per-row capture stores the **effective composite** rate **plus the leg references** (OD-P2-3-G) — the exposure's inputs are wholly inside the snapshot (the AD-014 single-bind) and every row is fully reconstructable.

### OD-P2-3-E — input source → **snapshot-only**
**Decision (the load-bearing AD-014 invariant).** The exposure binder reads its inputs **EXCLUSIVELY from the bound `dataset_snapshot`'s pinned components' captured content** (positions, valuations, FX) — the immutable serialized values (it may re-verify by re-resolution, but the **compute** consumes captured content). It **MUST NOT** call `reconstruct_subtree_holdings_as_of` / live `resolve_position`/`resolve_valuation` / `reconstruct_fx_rate_as_of` to source the compute. A snapshot is **MANDATORY** (the run's `input_snapshot_id` is non-null at the governed-write path). The snapshot is built with **`purpose=EXPOSURE_INPUT`** and **extended (P2-3) to pin FX** (`COMPONENT_KIND_FX` minted; **inside the snapshot package**, `build_snapshot` pins, for each distinct mark currency, the `convert`-path FX leg(s) to the run's base — using `reconstruct_fx_rate_as_of` at build to select each leg's as-of version; `_reresolve_content` gains an FX branch). **This is an ADDITIVE extension of the snapshot package — NOT a schema redesign** (the `dataset_snapshot`/`_component` tables are unchanged; `component_kind` is an un-CHECKed String; the P2-2 plan §10 pre-committed this P2-3 step). A scope-fence test forbids the **exposure** module from importing the live-read holdings/position/valuation/FX resolvers (the fence is on the *exposure* compute; the *snapshot* package's build-time pinning is exempt).

### OD-P2-3-F — `calculation_run` binding (FW-RUN §5 / TR-15)
**Decision.** Reuse the shipped `calculation_run` (ENT-026, status-mutable). The run binds: (1) **`code_version`** (mandatory — the deterministic-rollup anchor); (2) **`input_snapshot_id`** (non-null at the governed-write path); (3) `assumption_set_id` **N/A-with-rationale**; (4) `random_seed` NULL-with-"deterministic"; (5) scenario **N/A**; (6) `initiated_by` (the principal); (7) run timestamps + the **NEW additive `environment_id`** column (migration `0018`, on the status-mutable table — a non-breaking add). **`model_version_id` is N/A-with-recorded-rationale** ("no estimation model — deterministic captured-mark rollup"); **never mint a sham model_version**. Lifecycle: `create_run` (`CALC.RUN_CREATE`) → `update_run_status(RUNNING)` → compute → `update_run_status(COMPLETED|FAILED)` (`CALC.RUN_STATUS_CHANGE`).

**Failure model — split by timing (review fold — audit #4 / qa #7,#8; resolves "FAILED-with-outcome vs no-orphan-run").** There are TWO distinct failure classes, with DIFFERENT artifact expectations:
- **(a) Pre-create REFUSAL** — a missing FW-RUN prerequisite detected *before* `create_run` (null/absent `input_snapshot_id`, missing `code_version`, missing `environment_id`, missing `initiated_by`): the binder **raises before any run exists** ⇒ **ZERO `calculation_run`, ZERO `exposure_aggregate`, ZERO audit** ("no orphan run" in the literal sense — nothing was created).
- **(b) Post-create FAILED** — a gate that can only be evaluated *after* the run is RUNNING (snapshot incompleteness, a missing FX leg, a cross-tenant component): the binder marks the run **FAILED with `outcome='failure'`** and **commits the FAILED run + its `CALC.RUN_CREATE` + `CALC.RUN_STATUS_CHANGE(→FAILED)` audit**, but writes **ZERO `exposure_aggregate`**. The FAILED run is the durable evidence of the refused attempt (CTRL-032), **not** an orphan. "**No orphan run**" is therefore defined as: *no COMPLETED run without its result rows, and no run created when a prerequisite is absent.*
- **(c) Emit-path rollback** — if `record_event`/`record_lineage` itself raises during an otherwise-successful write, the whole governed unit rolls back co-transactionally (no `exposure_aggregate`, no COMPLETED run) — the P2-2 CTRL-032 negative test, mirrored.

**`outcome='failure'` requires the one additive `calc/service.py` touch** (review fold — audit #4 / qa #8): `update_run_status(...)` gains an optional **`outcome: str = "success"`** forwarded to `record_event`, defaulting so every existing `CALC.RUN_STATUS_CHANGE` caller is **behavior-unchanged** (the ingestion `.INGEST`-additive precedent). **`calc/service.py` is NOT frozen** — only `audit/service.py` is; this is the sole `calc/service.py` change. The taxonomy's doc-only `CALC.RUN_START/COMPLETE/FAIL` labels are an R-07 reconciliation item — **not silently invented**.

### OD-P2-3-G — `exposure_aggregate` shape
**Decision.** `exposure_aggregate` (IA, in `APPEND_ONLY_TABLES`):
- `id`, `tenant_id`, `system_from` (IA common columns).
- `calculation_run_id` (GUID FK → `calculation_run`, **NOT NULL**) — run-tracked.
- `input_snapshot_id` (GUID FK → `dataset_snapshot`, **NOT NULL**) — denormalized for the AD-014 gate + as-of queries (redundant with the run's, asserted equal).
- `portfolio_id`, `instrument_id` (the grain) + `base_currency`, `mark_currency` (ISO String(3)).
- **Captured inputs (for self-auditing + the deterministic recompute):** `signed_quantity` (Numeric(28,8), the position scale), `mark_value` (Numeric(20,6), the money scale).
- **FX capture (review fold — chief-arch #1 / data-arch #3):** `fx_rate` (Numeric(28,12)) = the **EFFECTIVE composite multiplier** `mark_currency → base_currency` (the product of the convert legs, HALF_UP-quantized to 12dp) — it equals a single published `fx_rate.rate` **only** in the direct/identity case; `fx_legs` (**JSON ordered list, NOT a hard FK**) = the pinned legs the conversion used, each `{fx_rate_version_id, snapshot_component_id, base, quote, rate}`. The **authoritative version-pin is the snapshot's `COMPONENT_KIND_FX` component(s)** (+ the lineage trail) — `exposure_aggregate` (an immutable IA row) **must not hold a hard FK to a supersedable FR `fx_rate` row** (the polymorphic version-pin convention of `dataset_snapshot_component`/`lineage_edge`). `fx_legs = []` and `fx_rate = 1` **only** for the identity case (`mark_currency == base_currency`).
- **`exposure_amount` (Numeric(28,6), the money scale — review fold — data-arch #2):** `= quantize_HALF_UP(signed_quantity × mark_value × fx_rate, 6)` (in `base_currency`; 6dp matches the `valuation.mark_value`/`cost_basis` money convention — it is a base-currency monetary value, not a unit count). The HALF_UP quantization is the P2-1 canonical-serialization precedent.
- `exposure_type` (String(30), controlled-vocab; **`MARKET_VALUE`** only in v1).
**NO** risk/VaR/ES/sensitivity/greek/scenario/return/weight column. **NO** `supersedes_id` (append-only; a re-run is a new run). **Rationale + the recompute contract (review fold — qa #9).** Carrying the inputs makes each row independently verifiable: the **self-audit invariant is exact by construction** — `exposure_amount == quantize_HALF_UP(signed_quantity × mark_value × fx_rate, 6)` computed from the **stored, rounded** `fx_rate` — while the **independent/determinism cross-check** (recomputing the effective rate from the pinned legs, and the Σ) asserts **TR-13 tolerance** (`ε_rel = 1e-12`), never exact float `==`. The Σ is auditable without re-reading the snapshot.

### OD-P2-3-H — audit
**Decision.** The run reuses the **shipped `CALC.RUN_CREATE` (create) + `CALC.RUN_STATUS_CHANGE` (per transition)** emitters from `calc/service.py` — extended only by the additive **`outcome` parameter** on `update_run_status` (see OD-P2-3-F; `outcome='failure'` on a **post-create FAILED** run, `'success'` default everywhere else). Per the OD-P2-3-F failure model: a **pre-create refusal emits ZERO audit** (no run exists); a **post-create FAILED run commits `CALC.RUN_CREATE` + `CALC.RUN_STATUS_CHANGE(→FAILED, outcome='failure')`** (the audited evidence of the refusal). **No new `EXPOSURE.*` audit category is minted** — the `exposure_aggregate` rows are **run-tracked + lineaged** (metadata of the already-audited run), the `ingestion_staged_record`/result precedent. (If per-result granular audit is later required, `EXPOSURE.AGGREGATE_CREATE` at a fresh **EVT-210** block is the R-07 reservation — recorded, not built.) **`audit/service.py` stays FROZEN** (the `outcome` change is in `calc/service.py`, which is not frozen). The DQ gates emit the shipped **`DATA.VALIDATE`**.

### OD-P2-3-I — entitlement
**Decision.** **Wire the seeded `exposure.aggregate.run`** (currently reserved-unwired) + **mint `exposure.view`** (both governed-write/read for the first derived number):
- `exposure.aggregate.run` (the run maker) → **`data_steward`** + **`risk_analyst_1l`** + `platform_admin`. (risk_analyst_1l = the 1L analyst who runs exposure; data_steward = the data maker; admin.)
- `exposure.view` → `risk_analyst_1l` + `risk_manager_2l` + `data_steward` + **`auditor_3l`** + `platform_admin`. **`auditor_3l` is INCLUDED in `.view`** — a governed derived OUTPUT is exactly what 3L oversight reviews (the `dq.result.view`/`lineage.view` precedent), unlike proprietary-identity SoD where auditor was excluded.
Deny-by-default `require_permission`; a parity test (`test_exposure_permissions_grants_as_ratified`) pins the sets. (The `auditor_3l`-in-`.view` inclusion is the one grant worth an explicit confirm — OD-P2-3-2.)

### OD-P2-3-J — lineage
**Decision.** **Generalize the P2-1 internal lineage writer** (it already re-resolves a non-`data_source` source via a local Core Table) to also root a **`calculation_run`** source: add `SOURCE_TYPE_CALCULATION_RUN='calculation_run'` + a `calculation_run` local Core-Table resolution (RLS-stamp `tenant_id` from the resolved run, never caller input), and **accept + stamp `lineage_edge.run_id`** (the resolved run's id) on calculation-run-sourced edges (review fold — lineage #6). Edges per run, with **named `edge_kind`s** (review fold — lineage #5 — `edge_kind` is a free `String(50)`; "DEP-LIN" is the *RTM requirement id*, NOT an edge_kind):
- `dataset_snapshot → calculation_run` — `source_type='data_snapshot'`, **`edge_kind=EDGE_KIND_DEPENDENCY` (`"DEPENDS_ON"`, minted additively — no migration)**, target = the run.
- `calculation_run → exposure_aggregate` — `source_type='calculation_run'`, **`edge_kind=EDGE_KIND_ORIGIN`**, `run_id` = the run, target = each result row.
So `exposure_aggregate → calculation_run → dataset_snapshot → {position, valuation, fx} components → inputs` is fully retrievable (BR-6/BR-13 / RTM **DEP-LIN**, CTRL-006/013). **No `lineage_edge` migration** (polymorphic, GUID `source_id`, the existing `run_id` column). **Not a framework rewrite** — one new source-type token + one `edge_kind` value + one local-Table arm.

### OD-P2-3-K — data quality
**Decision.** All gates **fail-closed BEFORE any `exposure_aggregate` INSERT** (CTRL-032 whole-run rollback), reusing the `run_quality_check` path + `DATA.VALIDATE`; the `(params, dataset)` **Protocol is UNTOUCHED** (caller-side gates, the P2-1 pattern):
1. **Snapshot completeness** — already enforced at `build_snapshot` (every non-zero bound position has a same-as-of mark); the exposure run re-asserts the snapshot is complete.
2. **FX completeness** — every distinct `mark_currency` in the bound valuations has a resolvable `convert`-path to `base_currency` as-of (the pinned FX); a gap → fail closed (no run result). (A `non_negative`/positive check on rates is already the P2-2 `RANGE` evaluator — reused if a rate is re-validated.)
3. **No missing valuation mark** — subsumed by (1); a position without a mark cannot contribute an exposure (fail closed, not silently zero).
4. **Cross-tenant fail-closed** — the snapshot + every component is resolved ONLY under the acting tenant's RLS (the P2-1 binding-integrity invariant); a foreign id resolves to zero rows → fail closed.
