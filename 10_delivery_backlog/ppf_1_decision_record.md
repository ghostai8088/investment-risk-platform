## Part 2 — Design decisions (PPF-1 scope)

### OD-PPF-1-A — The pure-private factor definition
A **segment** = a declared strategy/region grain (e.g. `US_BUYOUTS`), minted as an ordinary `factor` row in a NEW factor family **`PRIVATE`** (a `FACTOR_FAMILIES` vocab widening — the verifier confirmed registration refuses unknown families, `marketdata/factor.py:149-151`). The factor is *pure private by construction*: its return is the component of member instruments' desmoothed returns NOT explained by their public proxy (OD-C). MSCI's 17-segment table is the reference grain; the platform mints only the segments the tenant's book needs (demo: 2 — PRIVATE_CREDIT + PRIVATE_EQUITY). **Frequency honesty (verifier fold):** `FACTOR_FREQUENCIES` admits only DAILY today (`marketdata/models.py:178-180`) — registering the segment factor as DAILY would be a lie (its realizations are appraisal-period). PPF-1 mints **`FREQUENCY_APPRAISAL`** (irregular, interval-disclosed), admitted at registration ONLY for PRIVATE-family factors. The covariance/VaR DAILY gates are deliberately NOT touched — they will correctly refuse an APPRAISAL-frequency factor until PPF-2/3 mint the declared frequency-conversion convention, which is exactly the fail-closed composition the arc wants (no accidental mixing of appraisal-grain and daily-grain risk).

### OD-PPF-1-B — Segment membership (fork → OQ-PPF-1-4; verifier CLAIM 2 REFUTED the naive form — the fork survives WITH two named guards)
**Recommendation: a membership = a captured loading row in the (FL-1-widened) `proxy_mapping` table** — the private instrument carries one row onto its PRIVATE-family segment factor, `weight=1`, `mapping_method=MANUAL` — zero new mapping *schema*, full bitemporal/pinning machinery inherited. Membership is **explicit captured data, never derived from `asset_class`** (dodges the silent-skip trap: only instruments given a membership row are members; a member without a current REGRESSION proxy blend fails the segment run closed as a named gap, the P3-7 rule).

**The verifier REFUTED "byte-identical as-is"** — isolation is NOT free; it requires exactly these guards (all fail-closed today, no silent corruption exists in any path):
1. **Family-scope the exposure snapshot builder** — `build_factor_exposure_snapshot` (`snapshot/service.py:717-730`) pins ALL current-head `proxy_mapping` rows (tenant+instrument+open filters only). Unguarded, a PRIVATE membership row gets pinned into every PA-2/FL-1 build snapshot and **refuses every new run pre-create** at `_adjudicate_proxies` (`factor_service.py:384-388` — the factor is never in the caller's `factor_ids`, and the escape is closed by the family gate at `:343-347`). Fix: the builder's proxy-row query filters to the binder-admitted families (excludes PRIVATE).
2. **Split the capture-admission family set from `LOADING_FACTOR_FAMILIES`** — capture refuses PRIVATE-family rows today (`marketdata/proxy_mapping.py:356-361`), and the factor itself can't even be registered (`FACTOR_FAMILIES` has no PRIVATE, `marketdata/factor.py:149-151`). The naive fix — adding PRIVATE to `LOADING_FACTOR_FAMILIES` — would silently widen the FL-1 binder gate, the coverage gate (`_assert_full_coverage`, `factor_service.py:409-435`, where a membership row would mask a genuinely unloaded atom), and the RBSA candidate gate (`proxy_weight_service.py:287-291`), because that constant is **declared shared-verbatim by all three** (`marketdata/models.py:160-163`). PPF-1 therefore **deliberately overturns the shared-verbatim invariant in writing**: a capture-admission set ⊋ the binder/candidate sets (which stay byte-identical), with the models.py comment updated to record the split and why.
3. **Total-VaR is safe by METHOD, and PPF-1 makes that load-bearing dependency enforced**: `build_var_total_snapshot` pins REGRESSION-method rows only (`snapshot/service.py:2511`), so a MANUAL membership is never seen — but a REGRESSION-method membership would demand estimation evidence and refuse. Guard: capture-side validation that PRIVATE-family rows MUST be `MANUAL` (turning the convention into an invariant).

Disclosure: the `list_proxy_mappings` read surface will interleave membership rows with proxy-blend rows (non-numeric; a docs/UI disclosure line). Alternative fork (B): a new dedicated membership table — cleaner naming, no shared-constant surgery, but +1 migration + a duplicated bitemporal capture service. The recommendation stays **A**: the two guards are small, sharply locatable, and the fail-closed behavior of every current path means the blast radius of a mistake is a refused run, not a wrong number.

### OD-PPF-1-C — The pure-private return construction (the econometric core; verifier CLAIM 1 folds applied)
Per segment `s`, per appraisal period `t`:
1. Per member instrument `i`: `pp_i,t = desmoothed_i,t − Σ_f w_i,f · R_f,t` — the desmoothed return minus the proxy-implied return. Weights = the pinned current-head **REGRESSION** proxy blend (the var-total pin precedent, `snapshot/service.py:2504-2561`, single-cited-run adjudication — which also cleanly separates blends from MANUAL membership rows in PPF-1's own pins). `R_f,t` = the factor's **captured returns compounded over the half-open window `(period_start, period_end]`** — the PA-3 convention verbatim (`risk/proxy_weight_service.py:338-351` + the fail-closed per-period coverage gate at `:314-327`; NOT "daily" — the mechanism is cadence-agnostic and HG-1 already feeds it quarterly returns). **Verifier fold: `_compound`/`_build_design` are module-private — PPF-1 EXTRACTS them into a shared helper** (a mechanical, behavior-preserving extraction from PA-3's file, regression-proven byte-identical), rather than duplicating the convention.
2. Pool across members: `PP_s,t = pool({pp_i,t})` — v1 pooling convention = fork OQ-PPF-1-2. **Verifier fold — the pooling grid is NEW machinery, not PA-3 reuse** (PA-3 aligns factors onto ONE instrument's period grid; nothing aligns two members' grids): v1 declares **identical-interval pooling** — members' `(period_start, period_end]` intervals must match exactly to pool; any grid mismatch fails the segment run closed as a named gap (cross-calendar interpolation = a recorded v2). The demo grids satisfy this trivially at min_members=1.
**The intercept question** (fork OQ-PPF-1-3): `pp_i,t` as defined RETAINS the mean out-of-proxy return (the α — liquidity premium / private outperformance) in the factor realization, per MSCI ("the pure private components... important sources of risk **and return**"). The alternative (subtract the fitted intercept, leaving mean-zero residuals) makes the factor risk-only.
All inputs pinned (AD-014; the verifier proved each pin kind + serializer already exists — desmoothed periods carry `period_start`/`period_end` verbatim, `snapshot/serialize.py:460-461`, and the PA-3 consumer reconstructs intervals from pins alone with zero live reads): the member desmoothing runs (COMPLETED, per-period rows only — the summary row shares the table, the existing pin-builder filter is the precedent), the proxy blend, the factor returns. Deterministic; NO estimation beyond what upstream runs already governed. **Honest disclosure: this is two-step desmoothing** (desmooth → subtract), which Shepard (2014) shows is inferior to single-step joint estimation — recorded as the Bayesian v3, not hidden.
