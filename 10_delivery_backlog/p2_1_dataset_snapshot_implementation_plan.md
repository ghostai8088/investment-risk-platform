## §1 — Requirements included
- The **AD-014 reproducibility primitive**: an immutable record that **pins + captures the exact physical input versions** an as-of computation will consume, so a later `calculation_run` (P2-3) is reproducible.
- **`dataset_snapshot`** (header) + **`dataset_snapshot_component`** (per-input physical-version pin **+ captured value**) — new entities, **IA true-append-only**.
- A governed **`build_snapshot` binder** that resolves the as-of input set via the shipped **set-returning** holdings composers under tenant RLS, pins physical versions, **captures the canonical content**, computes an **app-side hash** (`audit/hashing.sha256_hex`), and writes under tenant RLS.
- **Snapshot lineage** (snapshot → component → pinned input) via a **new internal lineage writer**.
- A **completeness DQ gate** (caller-side, via the shipped `run_quality_check`; fail-closed) over the bound set.
- The **cross-tenant binding-integrity invariant** (proprietary components pin own-tenant versions only).
- `calculation_run` (ENT-026) **readiness** — the snapshot is shaped so `calculation_run.input_snapshot_id` can reference it; **no wiring**.

## §2 — Requirements excluded (strict)
No implementation; no `exposure_aggregate` / exposure calc / any derived number; no `calculation_run` wiring or `environment_id` add (→ P2-3); no FX / price / curve / benchmark / market-data ingestion; no factor model / risk / VaR / ES / pricing / valuation model; no reporting/dashboard; no P2-2…P2-6 work; no P3+; no `audit/service.py` change; no new DQ-evaluator (the `non_negative`/`staleness` additions are P2-2/P2-4).

## §3 — Proposed entities / modules
- New package **`irp_shared/snapshot/`** — `models.py`, `events.py` (the `SNAPSHOT.*` caller constants), `service.py` (`build_snapshot` + `verify_snapshot` + tenant-filtered resolvers), `serialize.py` (the canonical serialization), `__init__.py`. **One-way imports** `snapshot → {portfolio, position, valuation, holdings, lineage, dq, audit, db}`; **nothing imports `snapshot`** (leaf, like holdings) — import-direction test; **`snapshot/` imports NO `calc` symbol** (readiness never becomes wiring).
- New thin API module **`apps/backend/.../api/snapshots.py`** (create + read + verify).
- **Migration `0016_dataset_snapshot`**: the two tables + symmetric RLS + a **migration-local** `APPEND_ONLY_TABLES = ("dataset_snapshot","dataset_snapshot_component")` that applies one `*_append_only` BEFORE UPDATE OR DELETE trigger per table **reusing the existing `irp_prevent_mutation()` function** (the `0001` module tuple is UNTOUCHED). Head `0015_valuation → 0016_dataset_snapshot`.

## §4 — Temporal classification
**IA, TRUE append-only** for BOTH tables (`ImmutableAppendOnlyMixin`; `__temporal_class__ = IMMUTABLE_APPEND_ONLY`; in the **migration-local** `APPEND_ONLY_TABLES` → P0001 trigger + the ORM `event.listen("before_update"/"before_delete", _block_mutation)` guard raising the **shared `audit.models.AppendOnlyViolation`** — the **TRANSACTION exemplar**, NOT a new exception). A snapshot is a knowledge-time pin (TR-09): created once, **never mutated**; a new input set ⇒ a **new** snapshot. **Explicitly NOT** the status-mutable `calculation_run`/`ingestion_batch` IA flavor (no in-place status). No `valid_*` axis.

## §7 — Component canonicalization & hash design (settles decision 1)
**Algorithm: SHA-256 hex via the shipped `audit/hashing.sha256_hex(canonicalize(...))`** (the audited, engine-independent primitive — `ensure_ascii=False`), computed in **application code** (`serialize.py`), **never in the DB** (engine-independent across the AD-011 SQLite/PG split).
**`captured_content`** = an ordered list of `[field, normalized_value]` pairs (per-kind fixed order). Normalization (so no value ever reaches `canonicalize`'s `default=str` fallback): **Decimal** → fixed-scale string at the column scale (`quantity` 8; `mark_value`/`cost_basis` 6) (QS-01/03); **datetime** → ISO-8601 UTC µs (QS-12); **date** → `YYYY-MM-DD`; **GUID** → lowercase 36-char; **null** → a dedicated sentinel.
**Per-kind field lists** (the immutable captured content):
- `POSITION`: `[id, tenant_id, portfolio_id, instrument_id, quantity, cost_basis, quantity_unit, position_source, restatement_reason, supersedes_id, valid_from, system_from, record_version]`.
- `VALUATION`: `[id, tenant_id, portfolio_id, instrument_id, valuation_date, mark_value, currency_code, mark_source, price_basis, restatement_reason, supersedes_id, valid_from, system_from, record_version]`.
- `PORTFOLIO` (EV): `[id, tenant_id, code, name, node_type, parent_portfolio_id, base_currency_code, status, description, valid_from, record_version]` (no `system_from`; **includes `name`/`description`** so the hash moves on any EV amend — though `record_version` is the authoritative drift signal, §8).
- **Excluded from the hash:** `valid_to` / `system_to` (mutable close-out markers — FR rows are close-out-UPDATEd, content immutable) + `created_at`/`updated_at`. *(restatement_reason/supersedes_id are INCLUDED — they are write-once provenance of the captured version.)*
**`manifest_hash`** = `sha256_hex(canonicalize([header_fields…, component_count, ordered_component_hashes]))` where header_fields = `(tenant_id, as_of_valid_at, as_of_known_at, as_of_valuation_date, binding_predicate_version)` and the component hashes are sorted by the **NORMALIZED (lowercase)** `(component_kind, target_entity_id)`. Determinism is **conditional on identical `(as_of_valid_at, as_of_known_at, as_of_valuation_date)`** — the rebuild test pins all three explicitly (the default-now `known_at` is not relied on). A **zero-component snapshot is rejected** by the completeness gate (no empty snapshot).

## §8 — Physical-version pinning + reproducibility semantics (the core guarantee)
- **The component captures the value (`captured_content`) AND pins the physical version (`target_entity_id` = the row surrogate `id` + coords).** Reproduction therefore does **not** depend on re-fetching the live row — the snapshot is self-sufficient (the parent-plan directive; closes the FR-content-immutability-is-service-enforced-not-trigger-enforced gap).
- **FR inputs (position, valuation) — byte-stable (the strong guarantee):** the binder resolves the as-of version via the set-returning enumerators (§9), pins `id + valid_from + system_from + record_version`, and stores `captured_content`. Because each FR version is a distinct row whose business content is never mutated (only `valid_to`/`system_to` close out, excluded from the hash), `verify` re-resolving by `(tenant_id, id)` via **`resolve_position`/`resolve_valuation`** (the explicit-tenant-predicate id resolvers — **never a bare `session.get(id)`**, so the cross-tenant predicate holds on SQLite too) + re-serializing + comparing to `captured_content` is **byte-stable** under any later supersede/correction (TR-09).
- **EV input (portfolio) — value-captured + drift-detected:** the snapshot stores the portfolio's `captured_content`, so it is **reproducible from the component** even though EV is amended in place. `verify` re-resolves via **`resolve_portfolio`** and reports **drift** when `record_version` (the authoritative signal — it bumps on ANY `_UPDATABLE` change incl. name/description) **or** `captured_content` differs. (EV cannot be *reconstructed* from the live table after an in-place amend — but it does not need to be, because the value is captured.)
- **The reconstruct-as-of re-check is an OPTIONAL non-authoritative diagnostic only.** At a microsecond boundary (a correction landing exactly at `as_of_known_at`), the half-open predicate (`system_to > known`) would return the corrected row, so a `reconstruct_*_as_of(as_of_valid_at, as_of_known_at)==pinned id` assertion is NOT a green-gate. The **authoritative** verify path is strictly: re-resolve by id (acting_tenant) + re-serialize + compare `content_hash`.

## §9 — Relationship to P1C (portfolio, position, valuation, holdings) + the enumerators
- **Reuses, never re-derives, never computes.** The binder enumerates the bound position set via the shipped **`reconstruct_subtree_holdings_as_of(acting_tenant, portfolio_id, as_of_valid_at, as_of_known_at)`** (tenant-bounded, cycle-safe, **arithmetic-free**) and resolves marks via **`attach_marks_as_of(..., valuation_date=as_of_valuation_date)`** (which calls `reconstruct_valuation_as_of` per holding at the fixed date). The single-row `reconstruct_*`/`resolve_*` resolvers are used only for the per-component **pin/verify re-resolution**, not enumeration.
- **Holdings (P1C-5)** is the read-composition sibling; the snapshot **persists pins + captured values + hashes**, not a view. The subtree scope is read composition, **not ABAC enforcement** (anchor-not-enforce → P6+).

## §10 — Relationship to `calculation_run` / ENT-026 readiness (no wiring)
- The snapshot's `id` is the future referent of `calculation_run.input_snapshot_id` (already a nullable placeholder). P2-1 creates **no `calculation_run` row, adds no `environment_id`, computes no exposure**. Readiness is a doc/test (a snapshot resolves + verifies by id under tenant RLS), enforced by the §17 import fence (`snapshot/` imports no `calc` symbol). The AD-014 gate is enforced in **P2-3**.

## §11 — APIs (settles decisions 7 & 8)
Service-layer binder is the core; a thin governed API is **included** so the primitive is exercisable:
- `POST /snapshots` — create (body: `purpose`, `as_of_valid_at`, optional `as_of_known_at` (default-now), optional `as_of_valuation_date` (default `date(as_of_valid_at)`), the binding scope `portfolio_id` (+ `subtree`)). The binder **first resolves the scope via `resolve_portfolio`/`resolve_descendants`** (which RAISE `PortfolioNotVisible` on a cross-tenant/unknown id) before any enumeration. Returns the header + `manifest_hash`. Guarded by `snapshot.create`.
- `GET /snapshots/{id}` — header + components. Guarded by `snapshot.view`.
- `GET /snapshots/{id}/verify` — re-resolves each component under tenant RLS (via the id resolvers, frozen `as_of_known_at`), re-serializes, compares; returns `{ok, drifted_components[]}`; **emits NO audit event** even when drift is found. Guarded by `snapshot.view`.
- No PUT/PATCH/DELETE (append-only → 405).

## §14 — RLS behavior
- **Symmetric tenant-scoped** (`USING == WITH CHECK == own-tenant`, ENABLE+FORCE; migration `0016`, byte-for-byte the `0015` loop) for both tables; **NEVER hybrid** (the **closed 5-table hybrid set asserted unchanged**); **no BYPASSRLS app path** (the app role is the constrained `irp_app`; ops `BYPASSRLS` role per AD-015 is never the app path).
- **Cross-tenant binding-integrity (load-bearing, decision 6), TWO layers:** (a) the **explicit `acting_tenant` predicate inside the shipped resolvers** (`resolve_portfolio`/`resolve_descendants` RAISE `PortfolioNotVisible`; `reconstruct_*`/`resolve_*` return 0 rows / raise `*NotVisible`) — **the only layer on SQLite logic tests**; (b) **PG FORCE-RLS** as the backstop. The binder **resolves the scope first (raising on a foreign/unknown `portfolio_id`)**, then enumerates; a foreign proprietary target ⇒ fail closed, **whole-unit rollback, no snapshot** (CTRL-032); **`component_count == 0` ⇒ fail closed** (no empty/foreign-scope snapshot). `component.tenant_id` is server-stamped == header tenant.

## §15 — Lineage behavior (settles decision 9 — **a new internal writer**)
- The shipped `record_lineage()` is **data_source-only** and cannot root a snapshot-sourced edge. P2-1 adds a **new internal writer `record_internal_lineage`** that: (a) **after the binder flushes the `dataset_snapshot` header**, **re-resolves the snapshot by id under the acting tenant's RLS-scoped session** — via a **local `Core Table` reference** to `dataset_snapshot` (the `dq/service.py:49` `_DATA_SOURCE` precedent, so **lineage imports nothing from `snapshot`** and the one-way import rule holds) — and **stamps `lineage_edge.tenant_id` from the resolved row, never caller input**; (b) **raises a fail-closed `SnapshotNotVisible`-analog on a 0-row resolution** (mirroring `DataSourceNotVisible`); (c) **emits NO audit event** (the edge is metadata of the already-audited `SNAPSHOT.CREATE`). Each component roots **one `lineage_edge`**: `source_type = SOURCE_TYPE_DATA_SNAPSHOT` (**a new constant `'data_snapshot'` added to `lineage/models.py`** alongside `SOURCE_TYPE_DATA_SOURCE`), `source_id = snapshot.id` → `(target_entity_type, target_entity_id)`. **No `lineage_edge` migration.** (The meaningful cross-tenant guarantee lives on the TARGET resolvers, §14 — the self-sourced snapshot's lineage tenant-stamp is a WITH-CHECK backstop.)

## §16 — Data-quality behavior (settles decision 10 — caller-side gate via the shipped engine, Protocol untouched)
- **Mechanism (settled).** The binder computes the **`expected_keys` universe** = every position version returned by `reconstruct_subtree_holdings_as_of(scope, as_of_valid_at, as_of_known_at)` with non-zero quantity, each REQUIRING a same-`as_of_valuation_date` `VALUATION` component (the §20-D5 v1 rule, tied to `binding_predicate_version`). It then computes the **gap = expected − actual**, **encodes each MISSING expected key as a dataset row with a sentinel-null `present` column**, and calls the shipped **`run_quality_check`** with a **pre-registered per-tenant `DataQualityRule(rule_type=NOT_NULL, params={'column':'present'}, severity=ERROR)`** over that derived dataset — so a non-empty gap **fails closed via the EXISTING `not_null` evaluator** + `DATA.VALIDATE` + the no-silent-failure `DataQualityError`. **The `DQEvaluator(params, dataset)` Protocol + the two shipped evaluators are genuinely UNTOUCHED; no new evaluator.** The completeness `DataQualityRule` is **seeded per tenant** (incl. the SYNTHETIC fixture) as part of P2-1's governed setup.
- **Ordering + rollback.** Flush the snapshot + components, run the gate (writing a `data_quality_result` with `target_entity_id = snapshot.id`); on a gap raise `DataQualityError` so the **whole unit (snapshot + components + the flagged result + audit + lineage) rolls back together** (CTRL-032; the flagged result is acceptable to lose on rollback — fail-closed = no snapshot).
- An **empty bound scope** (0 expected) is **rejected** (no zero-component snapshot).

## §17 — Tests
- **Logic (SQLite):** binder builds a snapshot from portfolio+positions+valuations; **`manifest_hash` deterministic** (re-build over the SAME pinned `(as_of_valid_at, as_of_known_at, as_of_valuation_date)` ⇒ identical `manifest_hash`); per-kind `serialize.py` unit tests (Decimal scale, datetime µs, date, GUID lowercase, null sentinel, close-out-marker exclusion, `name`/`description` inclusion for EV); `captured_content` round-trips; `component_count` folded into the manifest; empty-component rejected; duplicate-target unique-constraint.
- **Immutability (split):** a **SQLite** ORM `before_update`/`before_delete` test raises `AppendOnlyViolation` on header AND component mutation/delete; a **PG `*_pg`** test that a raw UPDATE/DELETE under `irp_app` raises **P0001** (mirroring `test_lineage_edge_append_only_at_db`), both tables.
- **Temporal-reproducibility mutation test (headline):** build a snapshot; **as-known correct** a bound valuation (new physical version, later `system_from`); **re-resolve + verify the SAME snapshot** (authoritative path: `resolve_valuation`-by-id + re-serialize + `content_hash` byte-identical, `captured_content` unchanged); a **live** `reconstruct_valuation_as_of(now)` returns the corrected value (the contrast). Repeat for a position supersede. *(Any optional reconstruct re-check passes the FROZEN `(as_of_valid_at, as_of_known_at)`, never a fresh now — and is non-authoritative.)*
- **EV drift test:** amend a bound portfolio (a **name-only** and a **status** amend) → `verify` reports the portfolio component **drifted via `record_version`** (proving record_version catches an edit even where the hash subset might not), others ok.
- **Completeness gate:** a bound set missing a required mark → `build_snapshot` **fails closed** (`DataQualityError`); a flagged `data_quality_result` is produced and the **whole unit rolls back** (zero `dataset_snapshot` + `dataset_snapshot_component` + snapshot-sourced `lineage_edge` + `SNAPSHOT.CREATE` `audit_event` rows).
- **PG FORCE-RLS (`irp_app`):** snapshot + components readable only by the owning tenant; other tenant + no-context → **0 rows**; **cross-tenant binding negative on BOTH SQLite (explicit-predicate path) and PG (FORCE-RLS)** — a foreign `portfolio_id` ⇒ `PortfolioNotVisible` / fail closed, **whole-unit rollback** (the same zero-sibling-rows assertion); the per-tenant audit chain verifies; **closed-hybrid-set asserted unchanged**.
- **Lineage:** each component roots exactly one `data_snapshot`-sourced ORIGIN edge (the cross-tenant guarantee is attributed to the target resolvers, not the lineage writer).
