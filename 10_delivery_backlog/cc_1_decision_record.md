## Part 3 — Design (OD-CC-1-A…H)

### OD-A — Commitment grain: (portfolio_id, instrument_id) — the committing portfolio × the private-fund INSTRUMENT (Tier-3, OQ-1)

ENT-015 `commitment` binds **`portfolio_id` (the tenant book whose unfunded exposure it is) + `instrument_id` (the private fund committed to — an ordinary `instrument` row, `asset_class ∈ PRIVATE_*`, the PA-0 convention)**, mirroring position/valuation/transaction's two-FK shape. Fields: `committed_amount PreciseDecimal(20,6)` strictly positive; `currency_code String(3)`; `commitment_date Date` (the vintage anchor CC-2 needs); `vintage_year` DERIVED-not-stored (from `commitment_date` — no redundant column to drift); the four FR mixin columns + the manually-declared FR trio (`record_version`, `supersedes_id`, `restatement_reason` — the mixin does NOT provide them, the ProxyMapping shape); partial-unique current-row `(tenant_id, portfolio_id, instrument_id) WHERE valid_to IS NULL AND system_to IS NULL`. A commitment INCREASE (successive close) is an FR supersede carrying the new total; genuinely separate same-fund commitments (rare: distinct closes kept distinct) are a recorded v1 limitation — the one-current-row rule is deliberate. **Two chain rules the verifier pass forced (adversarial findings 1-2): (i) under the one-current-row invariant, `(portfolio_id, instrument_id)` IS the stable commitment identity — FR supersede/correct mints a NEW row id per version (the exemplar mechanics), so version-row ids are NOT stable linkage targets; ENT-016 events therefore key on the stable identity, never on a version-row FK (OD-C). (ii) `currency_code` is CHAIN-IMMUTABLE: supersede REFUSES a differing currency (fail-closed validator) and the correct path is structurally parameterless (no re-denomination field exists) — immutable events validate against "the commitment's currency", so a re-denominating supersede would strand already-captured events in an incoherent mixed-currency chain; a genuinely re-denominated commitment is a recorded v1 limitation — NO close-out op ships in v1 (the five-op set cannot end a chain), so a first-capture currency fat-finger is permanent for that pair; the chain-terminate op is the recorded v2 candidate (the adversarial-review fold — the draft's 'close out and re-capture' directed to a nonexistent path).** A supersede lowering `committed_amount` below the sum of captured calls is PERMITTED (OD-E defers all funded arithmetic to CC-2; the capture layer does not adjudicate economics — stated in the docs).
**Rejected: portfolio-node-FUND grain** (`portfolio.node_type='FUND'` models the tenant's OWN book structure, not the external fund; it would disconnect CC-2 from the instrument-grained proxy/desmoothing substrate ENT-019 binds to). **Rejected: instrument-only grain** (unfunded exposure is a portfolio-book fact — "whose commitment" must be first-class).
