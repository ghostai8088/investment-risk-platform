> **What P2-6 is — and is NOT.** P2-6 **captures** vendor benchmark/index definitions + constituents the way P2-2/P2-4/P2-5 captured FX rates / prices / curves: an **EV `benchmark` header** (the index identity — code/name/family/currency/methodology label) owning an **FR `benchmark_constituent` membership** (effective-dated `(instrument, weight)` rows; create / effective-dated supersede / as-known correction / `reconstruct_membership_as_of`), governed by `MARKET.BENCHMARK_*`/`BENCHMARK_CONSTITUENT_*` audit + VENDOR `data_source` lineage + a fail-closed DQ gate, under symmetric tenant RLS. ## 4. Temporal classification
**`benchmark` = EV / entity-versioned** (`record_version` drift discriminator; the `portfolio` / instrument-reference EV precedent; symmetric RLS; **NOT** append-only; canonical ENT-009 = EV) — versions when the *definition* changes (`MARKET.BENCHMARK_UPDATE`). **`benchmark_constituent` = FR / fully-reproducible / bitemporal** (`FullReproducibleMixin`; `__temporal_class__ = FULL_REPRODUCIBLE`; **NOT** in `APPEND_ONLY_TABLES`; close-out UPDATEs; content-immutability service-enforced + tested) — the `fx_rate`/`price_point`/`curve`-header protocol, captured/superseded/corrected **as a set** per `(benchmark, effective_date)`. The **membership-as-of is FR-reconstructable** (`reconstruct_membership_as_of(valid_at, known_at)` for a `(benchmark, effective_date)`). **EV-header + FR-detail is a novel composite** (the `curve` FR+IA novelty precedent) — OD-P2-6-B; OQ-P2-6-1 the all-FR alternative.

## 5. Benchmark metadata grain — **recommended: `(benchmark_code, benchmark_source)`**
**Recommendation + rationale.** The v1 EV current-identity key is **`(tenant_id, benchmark_code, benchmark_source)`** — one current `benchmark` per `(code, vendor)`; `benchmark_source` in the key (multi-vendor coexistence, the `price_source`/`curve_source` precedent). All promoted key columns DB-level **NOT NULL**. The EV `record_version` carries the definition's version history; each new EV version is independently traceable to one VENDOR `data_source` ORIGIN edge.

## 6. Benchmark constituent grain
**`(tenant_id, benchmark_id, instrument_id, effective_date) WHERE valid_to IS NULL AND system_to IS NULL`** — a 4-part FR current-head partial-unique; one open weight per (benchmark × instrument × effective date). Columns: `benchmark_id` (NOT-NULL FK → `benchmark.id`), `instrument_id` (NOT-NULL FK via `resolve_instrument`), `effective_date` (`Date`, the separate immutable logical key, §7), `weight` (`Numeric(20,12)`, §10), `constituent_currency` (nullable String(3)), `tenant_id` (RLS, server-stamped from the header). The FR axes version the membership; the `(benchmark, effective_date)` set is written/closed as one governed unit.

## 7. Benchmark level / return grain, if included — **DEFERRED**
**Out of P2-6** (OD-P2-6-K). `benchmark_level` (FR per-date index level) + `benchmark_return` (captured vendor-published period returns) require a **net-new canonical ENT id minted via the Part-3 ratification** (`p2_implementation_plan.md`:115/126 — "not a free build choice"), gated on the confirmed P3 dependency. P2-6 mints **no** new canonical id. When a later slice scopes them: captured vendor-published values only, FR, the same capture protocol — **NO return calculation from levels** (OQ-P2-6-9). A scope-fence asserts no `benchmark_level`/`benchmark_return` table or return-math symbol in P2-6.

## 8. Effective-date / valid-time convention
`effective_date` is a **separate immutable logical-key `Date`** (the membership/reconstitution date the constituent set is FOR), carried forward verbatim, **never mutated**, DISTINCT from the FR `valid_from` axis (the `curve_date`/`price_date` precedent). `valid_from`/`valid_to` (valid-time) + `system_from`/`system_to` (knowledge-time) version the *membership* for a fixed `(benchmark, effective_date)`. `reconstruct_membership_as_of(valid_at, known_at)` reads the membership set true-at `valid_at` as-known-at `known_at` for a `(benchmark, effective_date)` (the `reconstruct_*_as_of` shape).

## 9. Weight convention
`weight` = a **canonical DECIMAL fraction** (NOT percent, NOT bps): 5% → `0.050000000000`; 12.5bp → `0.001250000000`. Scale **`Numeric(20,12)`** (the `curve.point_value` rate-precision scale — sub-bp weight precision; NOT money scale). v1 DQ: a **non-negative sanity `RANGE` `[0, 1]`** per constituent (`{column:"weight", min:0, max:1}` — one name's weight 0…100%). **Weight-sum-to-≈1.0 completeness DEFERRED** (OQ-P2-6-4 — set-sum is meaningful only over an atomically-coherent set; FR per-name corrections make a transient Σ≠1 legitimate; long-only captured weights v1; signed/leveraged weights a future extension). **No performance/return/active-weight math** — `weight` is captured verbatim.

## 10. Currency convention
The `benchmark` header carries a captured **`benchmark_currency`** (ISO String(3), the index denomination), validated via the reused hybrid-aware `resolve_currency` (own OR SYSTEM). The constituent carries an optional nullable **`constituent_currency`** which, **when non-null, is likewise validated via `resolve_currency`** (fail-closed `CurrencyNotVisible` 404 — matching the §21 test; review fold — security LOW), else omitted. **NO conversion** in P2-6 — values are stored in native currency; any cross-currency transform is a *later calculation*. No `convert` import in the benchmark binder.

## 11. Source / provenance model
`benchmark_source` a controlled-vocab String label (e.g. `BLOOMBERG`/`SP_DJI`/`MSCI`/`FTSE`) — a **key** component — plus a **VENDOR `data_source` ORIGIN lineage edge** via a new `ensure_vendor_source(VENDOR_BENCHMARK)` (the `VENDOR_FX`/`VENDOR_PRICE`/`VENDOR_CURVE` sibling, module-local to `marketdata/benchmark.py`; REUSES `record_lineage` UNCHANGED): one edge per new `benchmark` EV version and one per captured membership-set version, **both targeting the `benchmark` row** (constituents covered transitively via `benchmark_id`). **The shipped `LineageEdge`/`record_lineage` carry no `effective_date`/metadata column** (review fold — lineage HIGH), so the `effective_date` is recoverable via the constituent rows, **NOT** stored on the edge; per-set-version edges share `target_entity_id` (the v1 limitation; a membership-version surrogate is the deferred lever — OQ-P2-6-10). **No market-data feed-ingestion pipeline / connector / scheduler** — captures arrive through the governed binder.

## 12. Relationship to instrument
`benchmark_constituent.instrument_id` is a **NOT-NULL FK** via the reused hybrid-aware `resolve_instrument` (the `price_point` precedent) — a constituent references a **mastered** instrument; cross-tenant fails closed; an unknown instrument → `InstrumentNotVisible` 404 (fail-closed, no write). v1 requires the instrument resolved; an **opaque constituent identifier** for not-yet-mastered names is **DEFERRED** (OQ-P2-6-3). A scope-fence asserts the FK + resolver path (no opaque free-text instrument ref in v1).

## 13. Relationship to `dataset_snapshot`
**READINESS ONLY** — P2-6 mints **NO `COMPONENT_KIND_BENCHMARK`** and adds none to `SNAPSHOT_COMPONENT_KINDS` (the shipped comment reserves `PRICE/CURVE/REFERENCE`; `BENCHMARK` is not named there — the P2-5 "comment not a constant" precision) and changes **NO `build_snapshot`/`_reresolve_content`**. A benchmark/membership component is pinned **only** when a future calculation consumes it (the P2-3 `COMPONENT_KIND_FX` precedent). **NO `dataset_snapshot` schema redesign** (the tables are untouched).

## 14. Relationship to `calculation_run`, if any — **expected: none in P2-6**
**NONE.** Captured benchmark data wires no `calculation_run`; `calc`/`exposure`/`snapshot` are **untouched**; `exposure_aggregate` is **unchanged**; **no exposure recalculation**. ## 20. Data-quality behavior
Two layers, both fail-closed:
- **Binder-side value checks (`BenchmarkValueError` → 422, before any write; NOT the DQ Protocol):** out-of-range/garbage field shapes; the currency/instrument resolution (unknown → 404, fail-closed). *(No `curve_type`↔`reference_key`-style conditional invariant — benchmark has no analogous discriminator pair; the membership-set atomicity is enforced by the one-governed-unit write.)*
- **Governed DQ gate (co-transactional; `DATA.VALIDATE`; CTRL-032 rollback):** **required-field NOT_NULL** (header: `benchmark_code`/`benchmark_source`/`benchmark_currency`; each constituent: `benchmark_id`/`instrument_id`/`effective_date`/`weight`) + a **weight sanity `RANGE`** `{column:"weight", min:0, max:1}` over the constituent set, run via `run_quality_check` (the `price.py`/`curve.py` `dataset=[{…}]` pattern; one ordinary single-column RANGE rule — **simpler than `curve`'s value-type-conditional split**). The `(params, dataset)` **Protocol is UNTOUCHED** (no new evaluator). **Weight-sum-completeness (Σ≈1.0) + stale-membership gates DEFERRED** (OQ-P2-6-8 — set-grained completeness is a later slice).

## 21. Tests
SQLite logic + PG FORCE-RLS + endpoint + parity (the `fx_rate`/`price_point`/`curve` test shape):
- **EV metadata (positive):** capture a `benchmark` → `update_benchmark` bumps `record_version` (definition change); the `(benchmark_code, benchmark_source)` current-identity key (one current per `(code, vendor)`; two `benchmark_source`s = two benchmarks); `MARKET.BENCHMARK_CREATE`/`BENCHMARK_UPDATE` audited.
- **FR membership protocol (positive):** `capture_membership` (a full `(instrument, weight)` set for an `effective_date`) → `reconstruct_membership_as_of` (both axes: a valid-time supersede + an as-known correction reconstruct the right set); the 4-part current-head partial-unique (one open weight per (benchmark, instrument, effective_date); many `effective_date`s coexist). **Content-immutability per-column (review fold — qa LOW; more delicate here than `curve` since the FR detail rows ARE the content, close-out is an UPDATE to `valid_to`/`system_to` on those very rows):** on the prior (now-closed) constituent rows assert `instrument_id` + `weight` + `effective_date` are **byte-identical** to pre-close and **ONLY** `valid_to`/`system_to` changed; and `reconstruct_membership_as_of` at the **old** `known_at` still returns the **original** weights (the `test_curve.py` content-immutability shape, adapted to the no-immutable-child FR detail).
- **Membership set atomicity:** a supersede/correction creates a FRESH set (closing the prior open set) written as one governed unit; whole-set rollback on any failure (no partial set, no orphan rows).
