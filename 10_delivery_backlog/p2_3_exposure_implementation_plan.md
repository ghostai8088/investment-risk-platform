## 4. Temporal classification
`exposure_aggregate` = **IA, TRUE append-only** (`ImmutableAppendOnlyMixin`; `__temporal_class__ = IMMUTABLE_APPEND_ONLY`; in `APPEND_ONLY_TABLES` → P0001 trigger + ORM guard). `calculation_run` = **IA status-mutable** (UNCHANGED — status projects via `update_run_status`; history in the audit chain; NOT in `APPEND_ONLY_TABLES`).

## 5. `exposure_aggregate` grain — **recommended: `(portfolio_id, instrument_id, base_currency)`**
**Recommendation + rationale.** v1 = the **per-holding market-value exposure in base currency**: one row per `(portfolio, instrument)` of the bound snapshot, `exposure_amount = signed_quantity × mark_value × fx(mark_currency→base)`. This is **minimal** (no cross-portfolio aggregation, no subtree rollup, no netting) yet **maximally auditable** — each row maps **1:1** to one snapshot POSITION + one VALUATION + the FX used, so lineage, the DQ gate, and the deterministic recompute (`exposure_amount == qty × mark × fx`) are crisp per row. A **portfolio/subtree TOTAL** is a deterministic Σ over rows — a coarser grain that is purely **additive** later (OQ-P2-3-1), so it is **deferred** rather than guessed. (`exposure_type = MARKET_VALUE`, controlled-vocab; signed market value v1, OQ-P2-3-4.)

## 6. `calculation_run` binding model
`run_type='EXPOSURE_AGGREGATE'`; `initiated_by` = the principal; `input_snapshot_id` = the bound snapshot (**non-null at the governed-write path**); `code_version` = the deterministic anchor (**mandatory**); the **NEW additive `environment_id`** (migration `0018`); `model_version_id`/`assumption_set_id`/`random_seed`/scenario **N/A-with-recorded-rationale** (model-less deterministic rollup). Lifecycle: `create_run` (`CALC.RUN_CREATE`) → `update_run_status(RUNNING)` → bind+gate+compute → `update_run_status(COMPLETED)` — `CALC.RUN_STATUS_CHANGE`.

**TR-15 gate + the failure model split by timing (review fold — resolves the "FAILED-with-outcome vs no-orphan-run" ambiguity):**
- **Pre-create REFUSAL** — a missing FW-RUN prerequisite (null/absent `input_snapshot_id`, missing `code_version`/`environment_id`/`initiated_by`) detected **before `create_run`** ⇒ **raise; ZERO `calculation_run`, ZERO `exposure_aggregate`, ZERO audit** (nothing created — "no orphan run" literally).
- **Post-create FAILED** — a gate evaluable only after RUNNING (snapshot incompleteness / missing FX leg / cross-tenant component) ⇒ `update_run_status(FAILED, outcome='failure')`, **commit the FAILED run + its `CALC.RUN_CREATE` + `CALC.RUN_STATUS_CHANGE(→FAILED)` audit**, **ZERO `exposure_aggregate`** (the FAILED run is durable refusal evidence, not an orphan).
- **"No orphan run"** ≡ no COMPLETED run without result rows + no run created on a missing prerequisite. (Emit-path failure → co-transactional CTRL-032 rollback, the P2-2 negative mirrored.)

## 7. `dataset_snapshot` binding model
The run binds exactly one `dataset_snapshot` (`purpose=EXPOSURE_INPUT`). **The exposure binder reads ONLY the snapshot's pinned components' captured content** (`list_components`) — POSITION (signed qty), VALUATION (mark + mark_currency), FX (the pinned leg rates + direction). It **MUST NOT** source the compute from live `reconstruct_*`/`resolve_*`/`reconstruct_fx_rate_as_of` reads (a scope-fence test forbids importing those into the **exposure** package). The snapshot is built (or supplied) with FX pinned (`COMPONENT_KIND_FX`, the P2-3 `build_snapshot` extension — where `reconstruct_fx_rate_as_of` legitimately selects each leg at build) so the AD-014 single-bind holds. The exposure run mode (build-in-request vs consume-existing-id) is **OQ-P2-3-3** (recommend: build-in-request, with a thin consume-existing variant).

## 8. FX conversion usage (single path — pure captured-content read)
For each bound `(position, valuation)` with a `mark_currency`: convert `mark_value` → `base_currency` by applying the P2-2 conversion **arithmetic** (direct → reciprocal → triangulation-through-base) **as a PURE function over the snapshot's pinned `COMPONENT_KIND_FX` captured values** — the exposure compute makes **no live `convert`/`reconstruct_fx_rate_as_of` call** (reproducibility = captured-content immutability; TR-09 — a re-run over the same snapshot is identical even after a later vendor correction). The FX legs were selected/pinned at **snapshot-build** time. **Exact-date** `rate_date == date(as_of_valid_at)` (v1); identity (`fx_rate=1`, `fx_legs=[]`) when `mark_currency == base`; **a missing leg at build → `FxRateNotFound` → the FX-completeness gate fails closed** (no snapshot ⇒ no exposure). The row stores the **effective composite** `fx_rate` (= the product of the legs, HALF_UP-quantized to 12dp; a published rate only in the direct/identity case) + **`fx_legs`** (the ordered pinned-leg references — NOT a hard FK; the snapshot `COMPONENT_KIND_FX` is the authoritative version-pin) — so a triangulated row is fully reconstructable. **No stale fallback, no silent 1.0, no interpolation.**

## 9. Base-currency convention
The base/reporting currency is a **run parameter** recorded on the run + every row's `base_currency`. Default: the bound **portfolio's `base_currency_code`** (existing EV field) if set, else **`USD`**. No new config store (deferred). All rows of a run share one `base_currency` (a multi-base report = multiple runs).

## 10. APIs (thin layer over `irp_shared.exposure`)
- **`POST /exposure/runs`** (gated **`exposure.aggregate.run`**; 201) — body `{portfolio_id, as_of_valid_at, base_currency?, as_of_known_at?, snapshot_id?}`. Builds (or binds the supplied) `EXPOSURE_INPUT` snapshot + creates a `calculation_run` + runs the gated compute; returns the run + the result set. Whole-unit rollback + the TR-15/DQ fail-closed mapping (409/422); cross-tenant/unknown → 404.
- **`GET /exposure/runs/{id}`** (gated **`exposure.view`**) — the run + its `exposure_aggregate` rows.
- **`GET /exposure/{id}`** (gated `exposure.view`) — a single `exposure_aggregate` row.
- **No PUT/PATCH/DELETE** (append-only).

## 11. Audit events
The run reuses the **shipped `CALC.RUN_CREATE` + `CALC.RUN_STATUS_CHANGE`** (+ the additive `update_run_status(outcome=…)` param — see §3/§6; `outcome='failure'` only on a **post-create FAILED** run, `'success'` default). Per the §6 failure model: a **pre-create refusal emits ZERO audit**; a **post-create FAILED run commits `CALC.RUN_CREATE` + `CALC.RUN_STATUS_CHANGE(→FAILED, outcome='failure')`**. The `exposure_aggregate` rows are run-tracked + lineaged (no per-result event; `EXPOSURE.AGGREGATE_CREATE` EVT-210 reserved-IF-later-needed). DQ gates emit `DATA.VALIDATE`. Per-tenant chain; DC-2 metadata only. **`audit/service.py` FROZEN** (the `outcome` change is in `calc/service.py`).

## 14. Lineage behavior
Per run, via the **generalized `record_internal_lineage`** (new `SOURCE_TYPE_CALCULATION_RUN`; **stamps `lineage_edge.run_id`**): a `dataset_snapshot → calculation_run` edge (**`edge_kind=EDGE_KIND_DEPENDENCY`** = `"DEPENDS_ON"`, minted additively — `edge_kind` is a free `String(50)`, no migration) + a `calculation_run → exposure_aggregate` edge per result row (**`EDGE_KIND_ORIGIN`**, `run_id` stamped). "DEP-LIN" is the **RTM requirement id**, not an `edge_kind`. `exposure_aggregate → run → snapshot → {position, valuation, fx} components` is fully retrievable. Tenant stamped from the RLS-resolved source. No `lineage_edge` migration; not a framework rewrite.

## 15. Data-quality behavior
Fail-closed **before any `exposure_aggregate` INSERT** (CTRL-032 whole-run rollback; `run_quality_check`/`DATA.VALIDATE`; Protocol untouched): **snapshot completeness** (P2-1, re-asserted), **FX completeness** (every mark currency has a pinned `convert`-path to base as-of), **no-missing-mark** (subsumed by completeness — a markless position fails closed, never silently zero), **cross-tenant fail-closed** (the binding-integrity invariant). No new generic evaluator required (caller-side gates over derived datasets, the P2-1 sentinel pattern; the P2-2 `RANGE` reused if a rate is re-validated).

## 16. Tests
SQLite logic + PG FORCE-RLS + endpoint + parity:
- **Reproducibility / correctness (positive):** build a snapshot → run exposure → assert each row's **self-audit invariant exact-by-construction** (`exposure_amount == quantize_HALF_UP(signed_qty × mark_value × fx_rate, 6)` from the **stored, rounded** `fx_rate`), and the **independent cross-check** (the effective `fx_rate` recomputed from the pinned `fx_legs`, and the set Σ) at **TR-13 tolerance** (`ε_rel = 1e-12`, never exact float `==`). Multi-currency: a USD mark + a EUR mark **triangulated to a JPY base** — assert the triangulated row's effective `fx_rate` **equals the product of its two pinned leg rates** re-resolved from the snapshot's `COMPONENT_KIND_FX` components, and `fx_legs` carries both leg refs. **Determinism** — same snapshot + `code_version` ⇒ identical results; **reproducible-under-FX-correction** — correct a bound `fx_rate` after the run, re-run over the SAME snapshot ⇒ identical exposure (captured FX content is reused; the exposure compute makes no live FX read; TR-09).
- **AD-014 / TR-15 fail-closed (the headline negatives — each row states EXACTLY which artifacts must/must-not exist):**
  - **Pre-create REFUSAL** (raise before any run): **null `input_snapshot_id`**, **missing `code_version`**, **missing `environment_id`**, **missing `initiator`** → ZERO `calculation_run` + ZERO `exposure_aggregate` + ZERO audit.
  - **Post-create FAILED** (gate after RUNNING): **incomplete snapshot** (a markless non-zero position), **missing FX leg** (`FxRateNotFound`), **cross-tenant** snapshot/portfolio component → a persisted **FAILED** `calculation_run` + `CALC.RUN_CREATE` + `CALC.RUN_STATUS_CHANGE(→FAILED, outcome='failure')` + **ZERO `exposure_aggregate`** (no COMPLETED run).
  - **Emit-path** (CTRL-032): monkeypatch `record_event`/`record_lineage` to raise on a good run → whole-unit rollback → ZERO `exposure_aggregate` + no COMPLETED run.
  Every row asserts ZERO result rows; gates run before any INSERT.
- **Input-source fence (load-bearing):** an AST/import scope-fence — the `exposure` compute imports **no** live-read resolver (`reconstruct_subtree_holdings_as_of`/`reconstruct_position_as_of`/`reconstruct_valuation_as_of`/**`reconstruct_fx_rate_as_of`**) — it reads the snapshot's captured content — and **no** risk/factor/covariance/vol/VaR/ES/scenario/pricing/return symbol + no P3 package; `ast.Mult` is **PERMITTED** (qty × mark × fx is the rollup). (The *snapshot* package's build-time FX pinning, which uses `reconstruct_fx_rate_as_of`, is exempt — the fence is on the *exposure* package.) Nothing imports `exposure`.
- **Append-only:** the P0001 trigger (PG, irp_app + positive control) + the ORM guard block UPDATE/DELETE on `exposure_aggregate`.
- **Audit:** `CALC.RUN_CREATE` (1) + `CALC.RUN_STATUS_CHANGE` (RUNNING + COMPLETED, `outcome='success'`) per successful run; a post-create FAILED run emits `CALC.RUN_STATUS_CHANGE(→FAILED, outcome='failure')`; a pre-create refusal emits **zero** audit; `verify_chain` ok (PG). The additive `update_run_status(outcome=…)` default-`'success'` keeps existing callers behavior-unchanged (a `calc/service.py` unit test). `audit/service.py` untouched.
- **Lineage:** the snapshot→run edge has `edge_kind == EDGE_KIND_DEPENDENCY` (`"DEPENDS_ON"`); the run→result edges have `edge_kind == EDGE_KIND_ORIGIN` **and `run_id IS NOT NULL`**; `exposure_aggregate` traces to the snapshot components; fail-closed CTRL-032 rollback (monkeypatch the emitter → zero rows).
- **Entitlement:** deny-by-default 403 (no DB side-effect on denial); the parity test; 404/409/422 mapping.
- **PG (`irp_app`):** tenant isolation + no-context → 0 rows; forged-tenant `WITH CHECK` 42501; **closed-hybrid-set-unchanged**; the append-only trigger.
- **Synthetic (OD-P2-L):** the P1C-6/P2-2 fixtures build → run → assert; **negative:** no synthetic module imports/writes `exposure_aggregate`/`calculation_run` + the SYNTHETIC tenant has ZERO such rows post-seed; `_SYN_MODULES` extended.
- **Migration:** `0018` applies (the `exposure_aggregate` table + the additive `calculation_run.environment_id`); `alembic check` drift-clean; downgrade `0018→0017→head` smoke (drop table + drop column). **Cross-slice:** `test_synthetic` `0018→0019`; any "exposure_aggregate must not exist" guard flipped (the P2-1/P2-2 fence-flip precedent — `exposure_aggregate` was in the still-future lists; remove it, keep the rest).

## 17. Acceptance criteria
A reproducible base-currency exposure for a snapshot, **fully run-bound + snapshot-gated**; each row self-auditing **exact-by-construction** (`exposure_amount == quantize_HALF_UP(qty × mark × fx_rate, 6)` from the stored rounded rate) with the independent recompute + Σ correct at **TR-13 tolerance** (1e-12); **no result exists without a complete TR-15 bind** (non-null snapshot + `code_version` + `environment_id` + the §5 metadata incl. recorded N/A); reproducible under a later FX/mark correction (captured content reused; no live FX read in the compute); fail-closed on any gate (pre-create refusal vs post-create FAILED per §6); symmetric FORCE-RLS isolation; `make check` + PG green; **no risk, no exposure outside a run/snapshot**.

## 18. Risks
- **Scope creep into risk** → mitigated by the vocabulary/import scope-fence + the "MARKET_VALUE only" `exposure_type` + the no-risk-symbol test.
- **Live-input leak** (reading un-snapshotted positions/valuations) → mitigated by the input-source import fence + the snapshot-only binder + reviewer focus.
- **FX/mark reproducibility** → mitigated by the snapshot-pinned, frozen-cutoff conversion + the under-correction re-run test.
- **`environment_id` semantics** (free value) → recorded as a run-environment label (deployment/region); not a security boundary.
- **Grain regret** → the per-holding atom is the most general (a total is a Σ); a coarser grain is additive (OQ-P2-3-1).
