# Master Build Prompt

You are supporting the development of a full-scope enterprise investment risk platform for asset managers expanding into private markets.

This is not an MVP, not a proof of concept, and not a throwaway prototype. The product must be designed as an enterprise-grade platform from day one.

The platform must support:

1. Portfolio and position management
2. Public asset data ingestion
3. Private asset data ingestion
4. Market data and pricing
5. Reference data and security master
6. Market risk analytics
7. Credit risk analytics
8. Counterparty risk analytics
9. Liquidity risk analytics
10. Scenario and stress testing
11. Limit monitoring
12. Breach and escalation workflow
13. First Line of Defense dashboard
14. Second Line of Defense dashboard
15. Model inventory
16. Model validation workflow
17. Data quality and reconciliation
18. Data lineage
19. Audit trail
20. Reporting
21. Administration and entitlements
22. Integration management
23. Security and compliance evidence
24. DevOps and operations

For every feature, provide:

1. Business purpose
2. User roles impacted
3. Workflow description
4. Data model impact
5. API impact
6. UI impact
7. Calculation logic, if applicable
8. Audit events
9. Data lineage requirements
10. Entitlement requirements
11. Tests required
12. Documentation updates
13. Known limitations
14. Enterprise hardening considerations

Do not create throwaway code. Favor modularity, testability, reproducibility, lineage, auditability, and future enterprise deployment.
